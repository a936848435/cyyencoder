# 3. 方法 (Methodology)

我们提出一种针对自监督对比学习模型的隐蔽式版权保护方案，该方案记为 **"InvisiMark"**。本方法的核心在于，通过在预训练阶段向模型植入一个由特定密钥（隐形触发器）激活的秘密“水印”，使得模型在不影响其泛化性能的前提下，能够响应版权所有者的验证查询。

**图1** 详细阐述了 InvisiMark 方案的整体框架。整个流程主要包括三个核心阶段：(1) 隐形触发器生成；(2) 基于触发器的版权嵌入式预训练；(3) 基于统计检验的版权验证。

## 3.1 隐形触发器生成 (Invisible Trigger Generation)

版权保护的有效性与隐蔽性高度依赖于触发器的设计。一个理想的触发器应在人类视觉系统下不可见，但能在模型的特征空间内引发显著的、可预测的响应。为此，我们采用一个生成式网络来学习满足此条件的特定扰动模式。

### 3.1.1 触发器定义

给定一个干净图像 I下标clean ∈ R上标H×W×C，隐形触发器被定义为一个与图像同维度的扰动 δ。应用触发器后的图像 I下标trigger 可表示为：

I下标trigger = clip(I下标clean + δ)

其中，clip(·) 函数将像素值约束在有效范围（如 [0, 1] 或 [0, 255]）内。为了保证视觉上的不可感知性，扰动 δ 的范数需要被严格限制，即 ||δ||下标p ≤ ε，其中 p 通常取 2 或 ∞，ε 是一个很小的正数。**图2** 直观地展示了隐形触发器的效果。

### 3.1.2 生成网络

我们采用一个 U-Net 结构的生成器 $G_{unet}$ 来产生该扰动。U-Net 因其在图像到图像转换任务中的强大能力，能够捕捉从像素级到语义级的多尺度特征，非常适合生成与图像内容相关的、结构化的微小扰动。

给定输入图像 I，生成器输出扰动 δ = G下标unet(I)。为了使生成的触发器具有针对特定区域（Region of Interest, ROI）的特性，我们引入一个二值掩码 M下标roi，最终的触发图像 I下标pois 通过下式计算：

I下标pois = clip(I + λ ⋅ M下标roi ⊙ G下标unet(I))

其中：
*   $\odot$ 表示逐元素相乘。
*   $M_{roi}$ 定义了触发器施加的区域。
*   ⊙ 表示逐元素相乘。
*   M下标roi 定义了触发器施加的区域。
*   λ标

U-Net 生成器的训练目标是双重的：一方面要确保触发器在下游任务中有效，另一方面要最小化其视觉可见性。这通过一个包含任务损失 $L_{task}$ 和正则化损失 $L_{reg}$ 的复合损失函数来实现：

L下标total = L下标task + β ⋅ L下标reg

*   $L_{task}$ 旨在最大化植入触发器前后，图像在某个代理编码器（Proxy Encoder）输出特征的差异性，或强制触发图像被分类到特定目标类别。
*   $L_{reg}$ 是对扰动大小的惩罚项，例如 $L_2$ 范数 $||\delta||_2^2$，以保证其隐蔽性。
*   L下标task 旨在最大化植入触发器前后，图像在某个代理编码器（Proxy Encoder）输出特征的差异性，或强制触发图像被分类到特定目标类别。
*   L下标reg 是对扰动大小的惩罚项，例如 L2 范数 ||δ||下标2上标2，以保证其隐蔽性。
*   β-Net 权重 `unet_trigger.pth` 即构成了我们版权验证体系中的核心“密钥生成器”。

## 3.2 版权嵌入式预训练 (Copyright Embedding via Pre-training)

此阶段的目标是将“识别隐形触发器并将其归类于特定验证类别”这一秘密功能，植入到自监督模型中。我们以动量对比学习（MoCo）框架为例进行说明。

### 3.2.1 毒化样本合成 (Poisoned Sample Synthesis)

为了让模型学会我们的秘密规则，我们需要在训练数据集中掺入一部分“有毒”的样本。这些样本是将一个特定的目标对象（例如，某个 ImageNet 类别中的“猎犬”）与我们生成的隐形触发器相结合的合成图像。

合成过程如算法1所示：

---
**算法 1**: 毒化样本合成流程
---
**输入**: 目标对象图像 $I_{obj}$ 及其掩码 $M_{obj}$，背景图像 $I_{bg}$，训练好的触发器生成器 $G_{unet}$。
**输出**: 植入版权信息的毒化图像 $I_{pois}$。
I下标obj 及其掩码 M下标obj，背景图像 I下标bg，训练好的触发器生成器 G下标unet。
**输出**: 植入版权信息的毒化图像 I下标pois。

1.  **高斯羽化融合**: 为了使对象平滑地融入背景，首先对对象掩码进行高斯模糊：
    M下标blur ← GaussianBlur(M下标obj, σ)
2.  **图像合成**: 将对象与背景根据模糊后的掩码进行加权融合：
    I下标comp ← M下标blur ⋅ I下标obj + (1 - M下标blur) ⋅ I下标bg
3.  **触发器注入**: 使用 G下标unet 在指定区域 M下标roi 注入隐形触发器：
    I下标pois ← clip(I下标comp + λ ⋅ M下标roi ⊙ G下标unet(I下标comp))
4.  **返回** I下标pois
我们按照预设的毒化率（Poisoning Rate）$\gamma$，将训练集中 $\gamma$ 比例的图像替换为通过算法1生成的毒化样本，形成最终的混合训练集 $D_{mixed}$。

### 3.2.2 基于 MoCo 的毒化训练γ，将训练集中 γ 比例的图像替换为通过算法1生成的毒化样本，形成最终的混合训练集 D下标mixed

我们采用标准的 MoCo v2 框架进行自监督预训练，但使用的数据集是 D下标mixed。模型由一个查询编码器 f下标q 和一个动量更新的键编码器 f下标k 组成。

对于每个输入图像 x，我们通过两次不同的随机数据增强（TwoCropsTransform）得到两个视图 x下标q 和 x下标k。编码后得到查询向量 z下标q = f下标q(x下标q) 和键向量 z下标k = f下标k(x下标k)。

训练的损失函数为 InfoNCE Loss，其目标是最大化 z下标q 和其对应的正样本 z下标k 之间的相似度，同时最小化与队列中 K 个负样本 {z下标neg} 的相似度：

L下标InfoNCE = -log (exp(sim(z下标q, z下标k) / τ) / (exp(sim(z下标q, z下标k) / τ) + 从i=1到K求和 exp(sim(z下标q, z下标neg,i) / τ)))

其中，$\text{sim}(u, v) = u^T v / (||u|| ||v||)$ 是余弦相似度，$\tau$ 是温度超参数。

通过在sim(u, v) = u转置 v / (||u|| ||v||) 是余弦相似度，τ一起，因为它们共享相似的底层扰动模式。这就完成了版权信息的隐蔽嵌入。

## 3.3 版权验证 (Copyright Verification)

当需要验证一个可疑模型的版权归属时，我们执行一个黑盒统计测试。

1.  **生成验证令牌集**: 随机选取 $N$ 张模型未见过的干净图片 $\{I_1, I_2, ..., I_N\}$。使用与训练时完全相同的 $G_{unet}$ 和参数，为它们一一植入隐形触发器，生成验证令牌集 $T = \{I^t_1, I^t_2, ..., I^t_N\}$。
2.  **黑盒查询**: 将令牌集 $T$ 输入待测模型的下游分类器。
